# Suite Web Architecture

Trezor Suite Web is a browser-based application that provides access to Trezor wallet functionality without requiring installation. It's built with modern web technologies and optimized for security and performance.

## Overview

Suite Web is accessible at [suite.trezor.io/web](https://suite.trezor.io/web) and runs entirely in the browser.

## Key Components

### Application Layer

<Tabs>
  <Tab title="@trezor/suite">
    **Core Application Package**
    
    The main Suite package provides:
    - React components and screens
    - Redux store and middleware
    - Business logic
    - Routing and navigation
    - Hooks and utilities
    
    ```typescript
    // Entry point: packages/suite-web/src/index.tsx
    import { Suite } from '@trezor/suite';
    import { render } from 'react-dom';
    
    render(
      <Suite />,
      document.getElementById('app')
    );
    ```
    
    <Note>
      The same `@trezor/suite` package is used in both Web and Desktop builds.
    </Note>
  </Tab>

  <Tab title="@trezor/connect-web">
    **Browser-Optimized Connect**
    
    Web-specific implementation of Trezor Connect:
    - Loaded as part of JavaScript bundle
    - WebUSB for device communication
    - Browser-compatible crypto
    
    ```typescript
    import TrezorConnect from '@trezor/connect-web';
    
    // Initialize Connect
    TrezorConnect.init({
      manifest: {
        email: 'info@trezor.io',
        appUrl: 'https://suite.trezor.io/web',
      },
    });
    
    // Use WebUSB to communicate
    const result = await TrezorConnect.getAddress({
      path: "m/49'/0'/0'/0/0",
      coin: 'btc',
    });
    ```
    
    <Info>
      Webpack replaces `@trezor/connect` imports with `@trezor/connect-web` in the web build. See [webpack config](https://github.com/trezor/trezor-suite/blob/develop/packages/suite-build/configs/web.webpack.config.ts).
    </Info>
  </Tab>

  <Tab title="State Management">
    **Redux Toolkit Store**
    
    Centralized state management:
    - **Slices**: Modular state containers
    - **Thunks**: Async actions
    - **Selectors**: Derived state
    - **Middleware**: Side effects
    
    ```typescript
    // Store structure
    {
      suite: { ... },           // App state
      wallet: { ... },          // Wallet data
      devices: [ ... ],         // Connected devices
      accounts: [ ... ],        // User accounts
      transactions: { ... },    // Transaction history
      fiat: { ... },           // Fiat rates
      ... 
    }
    ```
    
    **Persistence**: State saved to IndexedDB
  </Tab>

  <Tab title="Routing">
    **React Router v6**
    
    Client-side routing with:
    - Code splitting per route
    - Lazy loading
    - Protected routes
    - Redirect handling
    
    ```typescript
    const routes = [
      { path: '/', component: Dashboard },
      { path: '/accounts', component: Accounts },
      { path: '/accounts/:symbol/:accountIndex', component: AccountDetail },
      { path: '/settings', component: Settings },
      // ... more routes
    ];
    ```
  </Tab>
</Tabs>

## Key Differences: Web vs Desktop

<Tabs>
  <Tab title="Connect Integration">
    ### Web
    ```typescript
    // @trezor/connect-web is part of bundle
    import TrezorConnect from '@trezor/connect';
    // ↑ Webpack replaces with '@trezor/connect-web'
    
    // Direct WebUSB communication
    const result = await TrezorConnect.getPublicKey(...);
    ```
    
    ### Desktop
    ```typescript
    // @trezor/ipc-proxy in renderer
    import TrezorConnect from '@trezor/connect';
    // ↑ Proxies to main process via IPC
    
    // Main process has real Connect with USB/HID
    const result = await TrezorConnect.getPublicKey(...);
    ```
    
    <Note>
      Web uses direct WebUSB; Desktop uses IPC to Node.js main process.
    </Note>
  </Tab>

  <Tab title="Firmware Updates">
    ### Web
    - Firmware downloaded from **CDN**
    - URL: `https://suite.trezor.io/web/build/static/connect/data/firmware/[version]/[file]`
    - Uses `fetch()` API
    - **Requires internet**
    
    ```typescript
    // Fetched on-demand
    const firmwareUrl = `${CDN_URL}/firmware/2.6.0/trezor-2.6.0.bin`;
    const response = await fetch(firmwareUrl);
    const firmware = await response.arrayBuffer();
    ```
    
    ### Desktop
    - Firmware **bundled** with app
    - Located in `bin/` directory
    - Uses `fs.readFile()`
    - **Works offline**
    
    ```typescript
    // Read from filesystem
    const firmwarePath = path.join(resourcesPath, 'bin/firmware/2.6.0/trezor-2.6.0.bin');
    const firmware = fs.readFileSync(firmwarePath);
    ```
  </Tab>

  <Tab title="Storage">
    ### Web
    - **IndexedDB**: Primary storage
    - **LocalStorage**: Small settings
    - **SessionStorage**: Temporary data
    - Domain-specific (separate per URL)
    
    ```typescript
    // IndexedDB for wallet data
    const db = await openDB('trezor-suite', 1);
    await db.put('accounts', accountData);
    ```
    
    ### Desktop
    - **IndexedDB**: Same as web
    - **File system**: Logs, exports
    - Stored in app data directory
    - Environment-specific paths
  </Tab>

  <Tab title="Limitations">
    ### Web Limitations
    ❌ **No Tor support**: Browser limitations
    
    ❌ **No Bluetooth**: WebBluetooth not implemented
    
    ❌ **Browser-dependent**: WebUSB only in Chromium browsers
    
    ❌ **Internet required**: Can't work fully offline
    
    ❌ **Limited file access**: No arbitrary file system access
    
    ### Desktop Advantages
    
    ✅ **Tor integration**: Built-in privacy
    
    ✅ **Bluetooth support**: BLE device connectivity
    
    ✅ **All browsers**: No WebUSB dependency
    
    ✅ **Offline mode**: Full functionality without internet
    
    ✅ **File system**: Export logs, backups, etc.
  </Tab>
</Tabs>

### Build Process

<Steps>
  <Step title="Development Build">
    ```bash
    yarn suite:dev
    ```
    
    - Webpack dev server on `http://localhost:8000`
    - Hot Module Replacement (HMR)
    - Source maps
    - Fast refresh
    
    **Experimental Vite:**
    ```bash
    yarn suite:dev:vite
    ```

      Vite is experimental for development only. Production uses Webpack.
    </Warning>
  </Step>

  <Step title="Production Build">
    ```bash
    yarn suite:build:web
    ```
    
    **Output**: `packages/suite-web/build/`
    
    Optimizations:
    - Minification (Terser)
    - Code splitting
    - Tree shaking
    - Asset optimization
    - Gzip/Brotli compression
  </Step>

  <Step title="Bundle Analysis">
    ```bash
    yarn workspace @trezor/suite-web analyze
    ```
    
    Opens Webpack Bundle Analyzer showing:
    - Bundle size breakdown
    - Module dependencies
    - Duplicate code
    - Optimization opportunities
  </Step>

  <Step title="Preview Production Build">
    ```bash
    yarn suite:build:web:preview
    ```
    
    Runs Vite preview server with:
    - Production build
    - Security headers applied
    - Localhost testing

</Steps>

## Security

### Content Security Policy (CSP)

Production builds include strict CSP headers:

```http
Content-Security-Policy: 
  default-src 'self';
  script-src 'self' 'wasm-unsafe-eval';
  style-src 'self' 'unsafe-inline';
  img-src 'self' data: https:;
  connect-src 'self' https://suite.trezor.io https://*.trezor.io wss://blockbook.trezor.io;
  font-src 'self' data:;
  object-src 'none';
  base-uri 'self';
  form-action 'none';
  frame-ancestors 'none';
```

<Info>
  CSP headers are configured in the CDN/hosting platform (Cloudflare, Nginx, etc.).
</Info>

### Other Security Headers

```http
X-Frame-Options: DENY
X-Content-Type-Options: nosniff
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin
Permissions-Policy: camera=(), microphone=(), geolocation=()
```

### HTTPS Only

<Warning>
  Suite Web **requires HTTPS**. WebUSB API is only available on secure origins.
</Warning>

## Browser Compatibility

### Supported Browsers

| Browser | Version | WebUSB | Notes |
|---------|---------|--------|-------|
| **Chrome** | 90+ | ✅ Yes | Recommended |
| **Edge** | 90+ | ✅ Yes | Chromium-based |
| **Brave** | 1.24+ | ✅ Yes | Privacy-focused |
| **Opera** | 76+ | ✅ Yes | Chromium-based |
| **Firefox** | 88+ | ❌ No | No WebUSB support |
| **Safari** | 14+ | ❌ No | Limited support |

<Note>
  **Recommended**: Chrome or Brave for full functionality.
</Note>

## Performance Optimization

<Tabs>
  <Tab title="Bundle Size">
    **Techniques**:
    - Code splitting by route
    - Lazy loading components
    - Tree shaking unused code
    - Dynamic imports
    
    ```typescript
    // Lazy load heavy components
    const AccountDetail = lazy(() => import('./screens/AccountDetail'));
    
    // Render with Suspense
    <Suspense fallback={<Loading />}>
      <AccountDetail />
    </Suspense>
    ```
    
    **Target sizes**:
    - Initial bundle: < 500 KB (gzipped)
    - Total app: < 5 MB (gzipped)
  </Tab>

  <Tab title="Loading Performance">
    **Strategies**:
    - Preload critical resources
    - Defer non-critical scripts
    - Optimize images (WebP)
    - Use service worker caching
    
    ```html
    <!-- Preload critical assets -->
    <link rel="preload" href="/static/css/main.css" as="style">
    <link rel="preload" href="/static/js/main.js" as="script">
    ```
  </Tab>

  <Tab title="Runtime Performance">
    **Optimizations**:
    - React.memo for expensive components
    - useMemo/useCallback hooks
    - Virtual scrolling for long lists
    - Debounce/throttle user input
    
    ```typescript
    // Memoize expensive computations
    const sortedTransactions = useMemo(
      () => transactions.sort(byDate),
      [transactions]
    );
    ```
  </Tab>

  <Tab title="Network">
    **Optimizations**:
    - HTTP/2 multiplexing
    - Brotli compression
    - CDN edge caching
    - WebSocket for blockchain data
    
    **Cache strategy**:
    ```http
    # Static assets
    Cache-Control: public, max-age=31536000, immutable
    
    # HTML
    Cache-Control: no-cache, must-revalidate
    ```
  </Tab>
</Tabs>

## Troubleshooting

<AccordionGroup>
  <Accordion title="WebUSB not working" icon="usb">
    **Possible causes**:
    1. Unsupported browser (use Chrome/Brave)
    2. Non-HTTPS connection
    3. Missing USB permissions
    4. Another app is using the device
    
    **Solutions**:
    ```javascript
    // Request permissions explicitly
    await navigator.usb.requestDevice({
      filters: [{ vendorId: 0x1209, productId: 0x53C1 }]
    });
    ```
  </Accordion>

  <Accordion title="Build errors" icon="triangle-exclamation">
    ```bash
    # Clean build
    rm -rf packages/suite-web/build
    rm -rf node_modules/.cache
    
    # Rebuild
    yarn build:essential
    yarn suite:build:web
    ```
  </Accordion>

  <Accordion title="Slow performance" icon="gauge">
    1. Check bundle size: `yarn suite:analyze`
    2. Profile with React DevTools
    3. Check network waterfall
    4. Verify CDN caching
  </Accordion>
</AccordionGroup>
