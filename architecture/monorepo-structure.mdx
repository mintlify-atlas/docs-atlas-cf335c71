# Monorepo Structure

Trezor Suite is organized as a monorepo using Yarn workspaces, containing multiple applications and shared packages. This structure enables code sharing, consistent tooling, and unified development workflows.

## Overview

The repository contains three main projects:

<CardGroup cols={3}>
  <Card title="Trezor Connect" icon="plug">
    Developer API for integrating Trezor into third-party applications
  </Card>
  <Card title="Trezor Suite" icon="desktop">
    Desktop and web application for managing Trezor hardware wallets
  </Card>
  <Card title="Suite Mobile" icon="mobile">
    Native mobile application for iOS and Android
  </Card>
</CardGroup>

## Workspace Configuration

The monorepo uses **Yarn 4.12.0** with workspace configuration defined in `package.json`:

```json
{
  "workspaces": {
    "packages": [
      "packages/*",
      "packages/connect-examples/*",
      "suite-native/*",
      "suite-common/*",
      "suite/*",
      "scripts"
    ]
  }
}
```

<Note>
  Node.js version **24** is required, managed via NVM. See the `.nvmrc` file in the repository root.
</Note>

## Build System

### Package Manager

**Yarn 4.12.0** (Berry) with:
- Workspace protocol for internal dependencies
- Plug'n'Play (PnP) for faster installs
- Constraints and policies for consistency

### Build Tools

<Tabs>
  <Tab title="TypeScript">
    **TypeScript 5.8.3** used throughout:
    
    - Project references for incremental builds
    - Strict type checking
    - Path aliases via `tsconfig.base.json`
    
    ```json
    {
      "compilerOptions": {
        "target": "ES2022",
        "module": "ESNext",
        "strict": true,
        "esModuleInterop": true
      }
    }
    ```
  </Tab>

  <Tab title="Bundlers">
    Different bundlers for different targets:
    
    **Webpack 5**: Suite Web & Desktop (production)
    - Code splitting
    - Tree shaking
    - Asset optimization
    
    **Vite**: Suite Web (experimental dev mode)
    - Faster dev server
    - ESM-based
    - Hot Module Replacement
    
    **Metro**: Suite Native
    - React Native bundler
    - Fast refresh
    - Platform-specific bundling
    
    **Electron Builder**: Desktop packaging
    - App signing
    - Auto-updates
    - Platform-specific builds
  </Tab>

  <Tab title="Testing">
    **Jest 29.7.0** for unit tests:
    ```bash
    yarn test:unit
    ```
    
    **Playwright** for E2E tests (Suite Web/Desktop):
    ```bash
    cd suite/e2e
    yarn test:e2e
    ```
    
    **Detox** for mobile E2E tests:
    ```bash
    yarn workspace @suite-native/app test:e2e
    ```
  </Tab>

  <Tab title="Nx">
    **Nx 22.4.3** for build orchestration:
    
    - Affected command detection
    - Computation caching
    - Parallel execution
    
    ```bash
    # Build only affected packages
    yarn nx:build:libs
    
    # Test only affected packages
    yarn nx:test-unit
    
    # Show affected projects
    yarn nx:show-affected
    ```
  </Tab>
</Tabs>

## Dependency Management

### Workspace Dependencies

Internal dependencies use the `workspace:*` protocol:

```json
{
  "dependencies": {
    "@trezor/connect": "workspace:*",
    "@suite-common/wallet-core": "workspace:*"
  }
}
```

### Version Resolutions

Critical packages are pinned in the root `package.json`:

```json
{
  "resolutions": {
    "typescript": "5.8.3",
    "react": "19.1.0",
    "react-native": "0.81.5",
    "electron": "40.1.0"
  }
}
```

<Warning>
  Changes to resolutions affect all packages. Test thoroughly before updating.
</Warning>

## Development Workflow

### Initial Setup

<Steps>
  <Step title="Clone Repository">
    ```bash
    git clone git@github.com:trezor/trezor-suite.git
    cd trezor-suite
    ```
  </Step>

  <Step title="Initialize Submodules">
    ```bash
    git submodule update --init --recursive
    git config --global submodule.recurse true  # Optional but recommended
    ```
  </Step>

  <Step title="Set Up Git LFS">
    ```bash
    git lfs install
    git lfs pull
    ```
  </Step>

  <Step title="Install Dependencies">
    ```bash
    nvm install  # Uses .nvmrc
    yarn         # Installs all workspace dependencies
    ```
  </Step>

  <Step title="Build Essential Packages">
    ```bash
    yarn build:essential
    ```
  </Step>
</Steps>

### Common Commands

<CodeGroup>
```bash Development
# Suite Web
yarn suite:dev

# Suite Desktop
yarn suite:dev:desktop

# Suite Mobile
yarn native:start
yarn native:android  # or yarn native:ios
```

```bash Building
# Build libraries
yarn build:libs

# Build production web
yarn suite:build:web

# Build desktop (platform-specific)
yarn workspace @trezor/suite-desktop build:mac
yarn workspace @trezor/suite-desktop build:win
yarn workspace @trezor/suite-desktop build:linux
```

```bash Testing & Linting
# Type check all packages
yarn type-check

# Run unit tests
yarn test:unit

# Lint JavaScript/TypeScript
yarn lint:js

# Lint styles
yarn lint:styles

```

```bash Maintenance

yarn deps

yarn update-submodules

yarn depcheck
```

</CodeGroup>

## Architecture Patterns

### Code Organization Principles

<AccordionGroup>
  <Accordion title="Feature-Based Organization" icon="folder-tree">
    Packages are organized by feature/domain rather than technical layer:
    
    ✅ Good:
    ```
    suite-native/
    ├── module-send/
    │   ├── components/
    │   ├── hooks/
    │   ├── screens/
    │   └── utils/
    ```
    
    ❌ Bad:
    ```
    suite-native/
    ├── components/
    ├── hooks/
    └── screens/
    ```
  </Accordion>

  <Accordion title="Dependency Direction" icon="arrow-right">
    Clear dependency hierarchy:
    
    ```
    suite-native/app
          ↓
    suite-native/module-*
          ↓
    suite-common/*
          ↓
    packages/*
    ```
    
    Lower-level packages never depend on higher-level ones.
  </Accordion>

  <Accordion title="Platform Separation" icon="layer-group">
    Platform-specific code is isolated:
    
    - `suite/*` - Desktop/Web only
    - `suite-native/*` - Mobile only
    - `suite-common/*` - Shared across platforms
    - `packages/*` - Platform-agnostic when possible
  </Accordion>

  <Accordion title="Naming Conventions" icon="signature">
    Consistent package naming:
    
    - `@trezor/*` - Core libraries, Connect, Suite Web/Desktop
    - `@suite-native/*` - Mobile-specific packages
    - `@suite-common/*` - Shared platform-agnostic code
    - `@suite/*` - Desktop/Web-specific utilities
  </Accordion>
</AccordionGroup>
