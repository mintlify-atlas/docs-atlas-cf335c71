# Suite Mobile Architecture

Trezor Suite Mobile is a React Native application that brings Trezor wallet management to iOS and Android devices. Built with Expo SDK, it provides portfolio tracking, device connectivity, and cryptocurrency management on the go.

## Overview

Suite Mobile combines native performance with cross-platform code sharing.

### Native Modules

<AccordionGroup>
  <Accordion title="USB Communication" icon="usb">
    **@trezor/react-native-usb**
    
    Native USB/HID transport:
    - Android: USB Host API
    - iOS: Not supported (cable limitations)
    
    ```typescript
    import { TrezorUSB } from '@trezor/react-native-usb';
    
    // Android only
    const devices = await TrezorUSB.enumerate();
    const session = await TrezorUSB.acquire(device);
    ```

  </Accordion>

  <Accordion title="Bluetooth Connectivity" icon="bluetooth">
    **react-native-ble-plx 3.5.0**
    
    BLE support for Trezor Safe 3+:
    - Device scanning
    - Pairing
    - Encrypted communication
    - Connection management
    
    ```typescript
    import { BleManager } from 'react-native-ble-plx';
    
    const manager = new BleManager();
    manager.startDeviceScan(null, null, (error, device) => {
      if (device?.name?.includes('Trezor')) {
        // Connect to device
      }
    });
    ```
    
    <Info>
      Bluetooth support available on both iOS and Android for compatible Trezor models.

  <Accordion title="Biometric Authentication" icon="fingerprint">

    Biometric security:
    - Touch ID / Face ID (iOS)
    - Fingerprint / Face unlock (Android)
    - Fallback to PIN
    
    ```typescript
    import * as LocalAuthentication from 'expo-local-authentication';
    
    const result = await LocalAuthentication.authenticateAsync({
      promptMessage: 'Authenticate to access wallet',
    ```

  </Accordion>

  <Accordion title="Secure Storage" icon="lock">

    Platform-specific secure storage:
    - iOS: Keychain
    - Android: EncryptedSharedPreferences

    ```typescript
      keychainAccessible: SecureStore.WHEN_UNLOCKED,
    });
    ```
  </Accordion>

## Development Setup

### Prerequisites

<Steps>
  <Step title="Install Node and Yarn">
    ```bash
    # Using NVM
    nvm install
    
    # Enable Yarn
    corepack enable
    ```
  </Step>

  <Step title="Install Platform Tools">
    <Tabs>
      <Tab title="Android">
        Follow [React Native Android setup](https://reactnative.dev/docs/set-up-your-environment?os=macos&platform=android):
        
        1. Install Android Studio
        2. Install Android SDK (API 34+)
        3. Configure `ANDROID_HOME` environment variable
        4. Install Java Development Kit (JDK 17)
        
        **Linux**: Install `watchman`
        ```bash
        # Ubuntu/Debian
        sudo apt-get install watchman
        ```
      </Tab>

      <Tab title="iOS">
        **macOS only**
        
        Follow [React Native iOS setup](https://reactnative.dev/docs/set-up-your-environment?os=macos&platform=ios):
        
        1. Install Xcode (latest version)
        2. Install Xcode Command Line Tools:
           ```bash
           xcode-select --install
           ```
        3. Install CocoaPods:
           ```bash
           sudo gem install cocoapods
           ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Clone and Setup">
    ```bash
    git clone git@github.com:trezor/trezor-suite.git
    cd trezor-suite
    
    # Initialize submodules
    git submodule update --init --recursive
    git lfs install && git lfs pull
    
    # Install dependencies
    yarn
    yarn build:essential
    ```
  </Step>

  <Step title="Prebuild Native Code">
    ```bash
    # Clean prebuild (first time or after native changes)
    yarn native:prebuild:clean
    
    # Or for specific platform
    yarn native:prebuild:clean --platform android
    yarn native:prebuild:clean --platform ios
    ```
    
    <Note>
      Run `yarn native:prebuild` (or `yarn p`) after pulling/rebasing if native code changed.
    </Note>
  </Step>
</Steps>

### Running the App

<Tabs>
  <Tab title="Android">
    <Steps>
      <Step title="Start Metro Bundler">
        ```bash
        yarn native:start
        # or shortcut
        yarn s
        ```
        
        Keep this running in a separate terminal.
      </Step>

      <Step title="Connect Device or Start Emulator">
        **Physical Device:**
        - Enable Developer Options
        - Enable USB Debugging
        - Connect via USB
        - Recommended: [ADB over WiFi](https://developer.android.com/studio/command-line/adb#connect-to-a-device-over-wi-fi-android-11+)
        
        **Emulator:**
        - Start from Android Studio
        - Or via command line:
          ```bash
          emulator -avd Pixel_5_API_34
          ```
      </Step>

      <Step title="Build and Run">
        ```bash
        yarn native:android
        # or shortcut
        yarn a
        ```
        
        First build takes ~10 minutes.
      </Step>

      <Step title="Reverse Ports">
        Enable communication with localhost services:
        ```bash
        yarn native:reverse-ports
        # or shortcut
        yarn ports
        ```
        
        Reverses:
        - 8081: Metro bundler
        - 21328: Trezor Bridge
        - 19121: Debug server
      </Step>
    </Steps>

    <Tip>
      Use ADB over WiFi to free USB port for connecting Trezor device.
    </Tip>
  </Tab>

  <Tab title="iOS">
    <Steps>
      <Step title="Start Metro Bundler">
        ```bash
        yarn native:start
        # or shortcut
        ```

      </Step>

        **Via CLI:**
        ```bash
        yarn native:ios
        # or shortcut
        yarn ios
        ```
        
        **Via Xcode** (for debugging native errors):
        ```bash
        # Open workspace
        ```

        Then hit ▶️ Run button in Xcode.

    </Steps>

## Architecture Patterns

### Component Structure

<Tabs>
  <Tab title="Atomic Design">
    UI components organized by complexity:
    
    **Atoms** (`@suite-native/atoms`):
    - Button, Text, Input
    - Icons, Badges
    - Basic UI primitives
    
    **Molecules**:
    - Card, ListItem
    - Form fields
    - Combined atoms
    
    **Organisms**:
    - Navigation bars
    - Bottom sheets
    - Complex components
    
    **Screens**:
    - Full-page layouts
    - Feature screens
    - In feature modules
    
    ```typescript
    import { Button, Text } from '@suite-native/atoms';
    
    <Button onPress={handlePress}>
      <Text>Send Bitcoin</Text>
    </Button>
    ```
  </Tab>

  <Tab title="Theme System">
    **@suite-native/theme**
    
    Unified theming with:
    - Color tokens
    - Typography scale
    - Spacing system
    - Component variants
    
    ```typescript
    import { useTheme } from '@suite-native/theme';
    
    const Component = () => {
      const { colors, spacing } = useTheme();
      
      return (
        <View style={{ 
          backgroundColor: colors.backgroundPrimary,
          padding: spacing.large 
        }}>
          {/* content */}
        </View>
      );
    };
    ```
  </Tab>

  <Tab title="Navigation">
    **@suite-native/navigation**
    
    Type-safe navigation:
    
    ```typescript
    // Define routes
    type RootStackParamList = {
      Home: undefined;
      AccountDetail: { accountKey: string };
      Send: { accountKey: string; address?: string };
    };
    
    // Use navigation
    import { useNavigation } from '@react-navigation/native';
    
    const navigation = useNavigation<RootStackNavigationProp>();
    navigation.navigate('AccountDetail', { accountKey: 'btc-0' });
    ```
  </Tab>

  <Tab title="Forms">
    **@suite-native/forms**
    
    Form handling with validation:
    
    ```typescript
    import { useForm } from '@suite-native/forms';
    
    const { values, errors, handleSubmit } = useForm({
      initialValues: { address: '', amount: '' },
      validate: (values) => {
        // Validation logic
      },
      onSubmit: (values) => {
        // Send transaction
      },
    });
    ```
  </Tab>
</Tabs>

## Key Features

### Device Connectivity

<AccordionGroup>
  <Accordion title="USB Connection (Android Only)" icon="usb">
    Native USB support using `@trezor/react-native-usb`:
    
    1. User connects Trezor via USB OTG adapter
    2. Android prompts for USB permission
    3. App establishes HID connection
    4. Communication via USB protocol
    
    ```typescript
    import { TrezorUSB } from '@trezor/react-native-usb';
    
    // Request USB permission
    const hasPermission = await TrezorUSB.requestPermission(device);
    
    // Acquire session
    const session = await TrezorUSB.acquire(device.path);
    ```
  </Accordion>

  <Accordion title="Bluetooth Connection" icon="bluetooth">
    BLE support for Trezor Safe 3 and later:
    
    **Features**:
    - Device scanning
    - Automatic pairing
    - Encrypted communication
    - Connection persistence
    
    **Flow**:
    1. Enable Bluetooth on phone
    2. Scan for Trezor devices
    3. Select device from list
    4. Pair and connect
    5. Maintain connection for operations
    
    ```typescript
    import { useBluetooth } from '@suite-native/bluetooth';
    
    const { devices, connect, isConnected } = useBluetooth();
    
    // Scan for devices
    const trezorDevices = devices.filter(d => 
      d.name?.includes('Trezor')
    );
    
    // Connect
    await connect(trezorDevices[0]);
    ```
    
    <Info>
      Bluetooth works on both iOS and Android for compatible Trezor models.
    </Info>
  </Accordion>

  <Accordion title="Portfolio Tracker Mode" icon="chart-line">
    Watch-only mode without device:
    
    **Features**:
    - Import accounts via XPUBs
    - View balances and transactions
    - Track portfolio value
    - No sending capability
    
    **Use cases**:
    - iOS users (no USB support)
    - Portfolio monitoring
    - Public account tracking
    
    ```typescript
    import { addWatchOnlyAccount } from '@suite-native/accounts';
    
    // Add account by XPUB
    dispatch(addWatchOnlyAccount({
      coin: 'btc',
      xpub: 'xpub6...',
    }));
    ```
  </Accordion>
</AccordionGroup>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build Fails" icon="triangle-exclamation">
    ```bash
    # Clean everything
    yarn native:prebuild:clean
    
    # Clear Metro cache
    yarn native:start --reset-cache
    
    # Clear Gradle cache (Android)
    cd suite-native/app/android
    ./gradlew clean
    
    # Clear Pods (iOS)
    cd suite-native/app/ios
    pod deintegrate
    pod install
    ```
  </Accordion>

  <Accordion title="Metro Crashes" icon="circle-exclamation">
    ```bash
    # Clear watchman
    watchman watch-del-all
    
    # Restart Metro
    yarn native:start --reset-cache
    ```
  </Accordion>

  <Accordion title="Device Not Detected" icon="mobile">
    **Android**:
    - Check USB debugging enabled
    - Run `adb devices` to verify connection
    - Try different USB cable
    
    **iOS**:
    - Trust computer on device
    - Check Developer Mode enabled (iOS 16+)
    - Rebuild with Xcode
  </Accordion>

  <Accordion title="Native Module Errors" icon="puzzle-piece">
    ```bash
    # Rebuild native code
    yarn native:prebuild:clean
    
    # Android
    yarn native:android
    
    # iOS - reinstall pods
    cd suite-native/app/ios
    pod install
    cd ../../..
    yarn native:ios
    ```
  </Accordion>
</AccordionGroup>

| Feature | Mobile | Desktop | Web |
|---------|--------|---------|-----|

| **Framework** | React Native | Electron | React |
| **Device Connection** | USB (Android), BT | USB/HID | WebUSB |
