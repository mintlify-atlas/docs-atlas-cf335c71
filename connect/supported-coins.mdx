## Overview

Trezor Connect supports a wide range of cryptocurrencies across multiple blockchain networks. This page provides an overview of supported coins and how coin definitions are maintained.

<Info>
  Coin support depends on both the firmware version of your Trezor device and the Connect library version. Always check the `support` field in coin definitions for device-specific compatibility.
</Info>

## Bitcoin and Bitcoin-like Coins

Trezor devices have extensive support for Bitcoin and UTXO-based cryptocurrencies:

### Bitcoin Networks

<Tabs>
  <Tab title="Mainnet">
    ```typescript
    const result = await TrezorConnect.getAddress({
      path: "m/84'/0'/0'/0/0",
      coin: 'btc',
      showOnTrezor: true,
    ```

    **Features:**
    - Native SegWit (Bech32)
    - SegWit P2SH
    - Legacy addresses
    - Taproot (BIP86)
    - Multisig
    
    **SLIP44:** 0

  <Tab title="Testnet">
    ```typescript

      path: "m/84'/1'/0'/0/0",
      coin: 'test',
      showOnTrezor: true,
    ```

    **Features:**
    - Full Bitcoin testnet support
    - All address types
    - Ideal for development
    
    **SLIP44:** 1
  </Tab>
  <Tab title="Regtest">
    ```typescript

      path: "m/84'/1'/0'/0/0",
    ```

    **Features:**
    - Local regression testing
    - No blockchain link required
    
    **SLIP44:** 1

## Coin Definition Pipeline

Coin definitions in Trezor Connect are maintained through a structured pipeline:

<Steps>
  <Step title="Source of Truth">
    All coin definitions originate from the [trezor-firmware repository](https://github.com/trezor/trezor-firmware/tree/main/common/defs) as JSON files.
  </Step>
  <Step title="Trezor Common">
    Definitions are exported to the read-only [trezor-common repository](https://github.com/trezor/trezor-common), which is included in Connect as a git submodule at `submodules/trezor-common`.
  </Step>
  <Step title="Connect Integration">
    The `coins.json` file in `@trezor/connect-data` is built from the submodule using the `cointool` script.
  </Step>
</Steps>

<Warning>
  **Do not manually edit** `@trezor/connect-data/files/coins.json`. Changes should be made in the firmware repository and synced through the official pipeline.
</Warning>

## Checking Coin Support

Use the `getCoinInfo` method to verify coin support and retrieve coin metadata:

```typescript
const coinInfo = await TrezorConnect.getCoinInfo({ coin: 'btc' });

if (coinInfo.success) {
  console.log('Coin name:', coinInfo.payload.name);
  console.log('Shortcut:', coinInfo.payload.shortcut);
  console.log('SLIP44:', coinInfo.payload.slip44);
  console.log('Support:', coinInfo.payload.support);
}
```

## Adding New Coins

To add support for a new coin:

<Steps>
  <Step title="Check Compatibility">
    Verify the coin is compatible with Trezor's supported blockchain types (UTXO, EVM, etc.)
  </Step>
  <Step title="Firmware Definition">
    Create a coin definition in the [trezor-firmware repository](https://github.com/trezor/trezor-firmware/tree/main/common/defs) following the specification format.
  </Step>
  <Step title="Enable Connect Support">
    Enable Connect support in the [support.json](https://github.com/trezor/trezor-firmware/blob/main/common/defs/support.json) configuration.
  </Step>
  <Step title="Update Submodule">
    After the coin is merged in firmware, update the `trezor-common` submodule:
    ```bash
    yarn update-submodules
    ```
  </Step>
  <Step title="Rebuild Coin Data">
    Regenerate the `coins.json` file:
    ```bash
    yarn update-coins
    ```
  </Step>
</Steps>

<Info>
  For detailed instructions on the coin addition process, see the [supported-coins documentation](https://github.com/trezor/trezor-suite/blob/develop/docs/packages/connect/supported-coins.md) in the repository.
</Info>

## API Methods by Blockchain

Different blockchains have specialized API methods:

<Tabs>
  <Tab title="Bitcoin">
    - `getAddress` - Get Bitcoin address
    - `getPublicKey` - Get xpub
    - `signTransaction` - Sign Bitcoin transaction
    - `composeTransaction` - Compose UTXO transaction
    - `getAccountInfo` - Get account balance and history
  </Tab>
  <Tab title="Ethereum">
    - `ethereumGetAddress` - Get Ethereum address
    - `ethereumGetPublicKey` - Get public key
    - `ethereumSignTransaction` - Sign transaction
    - `ethereumSignMessage` - Sign message (EIP-191)
    - `ethereumSignTypedData` - Sign typed data (EIP-712)
  </Tab>
  <Tab title="Cardano">
    - `cardanoGetAddress` - Get Cardano address
    - `cardanoGetPublicKey` - Get public key
    - `cardanoSignTransaction` - Sign transaction
    - `cardanoGetNativeScriptHash` - Get script hash
  </Tab>
  <Tab title="Solana">
    - `solanaGetAddress` - Get Solana address
    - `solanaGetPublicKey` - Get public key
    - `solanaSignTransaction` - Sign transaction
  </Tab>
</Tabs>
